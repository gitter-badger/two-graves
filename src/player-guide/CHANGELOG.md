### Making catalog a little more robust with a build preparation step

[f063a873f6de08a115ed52cfc7e1274d02c5522b](https://github.com/rwdalpe/two-graves/commit/f063a873f6de08a115ed52cfc7e1274d02c5522b)

Winslow Dalpe <rwdalpe@gmail.com>

2015-06-29T12:34:54-07:00

Since various transformers and parsers might handle the catalog
differently, we'll make certain transformations a little more robust
by surrounding them in a group that specifies a base URL so that
we always get the right files.

To determine what the base URL should be, we add a build step to
set the base URL to the current path to the source files, and then
everything that references those files can be relative from that.

### Enabling XML Catalog support and dynamic glossary

[f319b3e12675682fa82276eaa53d9c81475a3f17](https://github.com/rwdalpe/two-graves/commit/f319b3e12675682fa82276eaa53d9c81475a3f17)

Winslow Dalpe <rwdalpe@gmail.com>

2015-06-29T00:14:54-07:00

Utilizing my changes to jing to enable XML Catalog support.

This is necessary to enable a dynamic glossary as described in
http://www.sagehill.net/docbookxsl/GlossDatabase.html

I expect that I can use the catalog much more extensively, though
and that it will help with splitting out content that will be
shared between books into a separate repository in the future.

The glossary is obviously just for show at the moment and contains
nothing of value.

The EPUB build has not yet been updated to use the glossary.
